Markdown 功能
Docusaurus 使用 Markdown 作为其主要内容编写格式。

:::tip 学习 Markdown

你可以在10分钟内学习 Markdown。

:::

Docusaurus 使用现代工具帮助你创建交互式文档。

MDX 编译器将 Markdown 文件转换为 React 组件，并允许你在 Markdown 内容中使用 JSX。这让你可以轻松地在内容中穿插 React 组件，创造令人愉悦的学习体验。

:::tip 使用 MDX 演练场

MDX 演练场 是你最好的新朋友！

它是一个非常有用的调试工具，可以展示 MDX 编译器如何将 Markdown 转换为 React。

选项：选择正确的格式（MDX 或 CommonMark）以及 Docusaurus 使用的以下插件：remark-gfm、remark-directive、rehype-raw。

:::

MDX 与 CommonMark {/* #mdx-vs-commonmark */}
Docusaurus 使用 MDX 编译器将 .md 和 .mdx 文件都编译为 React 组件，但根据你的设置，语法解释可能有所不同。

MDX 编译器支持 2 种格式：

MDX 格式：一个强大的解析器，允许使用 JSX

CommonMark 格式：一个符合标准的 Markdown 解析器，不允许使用 JSX

默认情况下，由于历史原因，Docusaurus v3 对所有文件（包括 .md 文件）都使用 MDX 格式。

可以通过 siteConfig.markdown.format 设置或 mdx.format: md 前置内容来选择使用 CommonMark。

:::tip 如何使用 CommonMark

如果你计划使用 CommonMark，我们推荐使用 siteConfig.markdown.format: 'detect' 设置。将根据文件扩展名自动选择适当的格式：

.md 文件将使用 CommonMark 格式

.mdx 文件将使用 MDX 格式

:::

:::danger 实验性的 CommonMark 支持

CommonMark 支持是实验性的，目前存在一些限制。

:::

<details><summary>Markdown 是声明式的</summary>
有些人可能认为 Markdown 和 HTML 之间存在一一对应关系，例如 ![预览](/img/docusaurus.png) 将始终按原样变成 <img src="/img/docusaurus.png" alt="预览" />。然而，事实并非如此。

Markdown 语法 ![消息](url) 只是声明性地告诉 Docusaurus 需要在此处插入一张图片，但我们可能会做其他事情，比如将文件路径转换为 URL 路径，因此生成的标记可能与其他 Markdown 渲染器的输出不同，也与简单的转录成等效 JSX/HTML 代码不同。

通常，你只应假设标记的语义（``` 栅栏变为代码块；> 变为引用等），但不能假设实际的编译输出。

</details>
前置内容 {/* #front-matter */}
前置内容用于向你的 Markdown 文件添加元数据。所有内容插件都有自己的前置内容模式，并使用前置内容来丰富从内容或其他配置推断出的默认元数据。

前置内容位于文件的最顶部，由三个破折号 --- 包围。内容被解析为 YAML。

md
---
title: 我的文档标题
更多数据:
  - 可以作为
  - as: 对象
    or: 数组
---
:::info

每个官方插件的 API 文档列出了支持的属性：

文档前置内容

博客前置内容

页面前置内容

:::

:::tip 增强你的前置内容

使用 Markdown 配置 parseFrontMatter 函数来提供你自己的前置内容解析器，或增强默认解析器。

可以重用默认解析器，用你自己的自定义专有逻辑来包装它。这使得可以实现方便的前置内容转换、快捷方式，或使用 Docusaurus 插件不支持的、来自外部系统的前置内容进行集成。

js
export default {
  markdown: {
    // highlight-start
    parseFrontMatter: async (params) => {
      // 重用默认解析器
      const result = await params.defaultParseFrontMatter(params);

      // 处理前置内容描述占位符
      result.frontMatter.description =
        result.frontMatter.description?.replaceAll('{{MY_VAR}}', 'MY_VALUE');

      // 创建你自己的前置内容快捷方式
      if (result.frontMatter.i_do_not_want_docs_pagination) {
        result.frontMatter.pagination_prev = null;
        result.frontMatter.pagination_next = null;
      }

      // 重命名来自另一个系统的不支持的前置内容
      if (result.frontMatter.cms_seo_summary) {
        result.frontMatter.description = result.frontMatter.cms_seo_summary;
        delete result.frontMatter.cms_seo_summary;
      }

      return result;
    },
    // highlight-end
  },
};
:::
